# Project Overview
This workspace contains SQL exercises and learning materials for **Day 05** of the **Data Analytics Level III** course. The project focuses on mastering SQL filtering techniques, sorting, and pattern matching using various operators and clauses.

## Directory Structure
- **Act_1/**: Activity 1 exercises, organized into parts.
    - **Part_A/**: Exercises 01-05 (Basic filtering with `WHERE`, `AND`, `OR`).
    - **Part_B/**: Exercises 06-10 (Filtering with `BETWEEN`, `IN`, `LIKE`).
    - **Part_C/**: Exercises 11-15 (Sorting and limiting results).
- **Act_2/**: Activity 2 exercises (`sc1.sql` to `sc5.sql`), focusing on scenario-based problem solving.
- **LearningJournal.md**: Personal reflections and insights on the SQL concepts covered during the day.

## Key Technologies
- **SQL**: Standard SQL queries for data retrieval.
- **Data Entities**: The queries target tables such as `sales` and `inventory`.

## Building and Running
Currently, there is no automated runner for these scripts. They are intended to be executed in a SQL client or integrated development environment (IDE) against a pre-configured database.

## Development Conventions
- **Documentation**: Each `.sql` file begins with a comment explaining the query's objective.
- **Validation**: Many scripts include a comment indicating the expected number of returned rows (e.g., `-- Rows returned = 8`) to help verify correctness.
- **Style**: Standard SQL keywords are typically written in UPPERCASE for readability.
